This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**oakphylo2026** is a multi-pipeline analysis project for an oak phylogenetics book chapter (2026), comparing phylogenetic trees inferred from three genomic datasets for *Quercus* species. The work builds on Hipp et al. 2020 (*New Phytologist* 226: 1198–1212).

Three datasets feed all comparative analyses:

| Dataset | Source | Produced by |
|---------|--------|-------------|
| `empiricalRAD` | Empirical RAD-seq, reference-guided assembly | (external — ipyrad + RAxML) |
| `simRAD` | Simulated RAD-seq from whole-genome sequences | [`RADinitio/`](RADinitio/) → external RAxML |
| `reSeq` | Pseudoreference whole-genome resequencing | [`genome_snps_and_refs/`](genome_snps_and_refs/) → external RAxML |

## Repository Layout

| Folder | Role | Author |
|--------|------|--------|
| [`topologies/`](topologies/) | **Primary R analysis pipeline.** Loads RAxML trees from all three datasets, prunes to focal clades, runs MDS ordination, builds cophyloplots/consensus trees, summarizes clade support, identifies rogue taxa. See [`topologies/CLAUDE.md`](topologies/CLAUDE.md) for full pipeline documentation. | Andrew Hipp |
| [`genome_snps_and_refs/`](genome_snps_and_refs/) | Shell + Python pipeline for SNP calling and pseudoreference generation from whole-genome resequencing reads (steps 0–8), plus *f*-branch admixture analyses (steps 9a, 9b). Designed for SLURM (Indiana University HPC). | Drew Larson |
| [`RADinitio/`](RADinitio/) | RAD-seq simulation pipeline. Takes a reference genome and produces simulated RAD loci for downstream tree inference. | K. Althaus |
| [`fbranch_interp/`](fbranch_interp/) | R script summarizing *f*-branch (Dsuite) results from `genome_snps_and_refs/9*/` into per-section means and `FIG5_*` outputs. | Andrew Hipp |

## Data Flow Across Pipelines

```
genome_snps_and_refs/        → topologies/data/ReSeq_phylos/         (reSeq trees)
genome_snps_and_refs/9a,9b/  → fbranch_interp/data/                  (Dsuite f-branch results)
RADinitio/                   → topologies/data/simulated_RAD/        (simRAD trees)
(external ipyrad + RAxML)    → topologies/data/ref_RAxML_snps/       (empiricalRAD trees)
```

External RAxML / ipyrad runs sit between most upstream pipelines and `topologies/`. Tree files (`.tre`) and ipyrad stats are committed to `topologies/data/` and are **not** regenerated by anything in this repo.

## Working in This Repository

- The four sub-pipelines are independent and run in different environments:
  - `topologies/` and `fbranch_interp/` — local R, working directory set to the subfolder.
  - `genome_snps_and_refs/` — SLURM scripts referencing absolute paths under `/N/project/oakphylo2025/` on the Indiana University HPC. They are reproduction artifacts, not portable scripts. Do not rewrite paths or re-run locally without major adaptation.
  - `RADinitio/` — bash + Python; assumes `radinitio` CLI on PATH.
- Each subfolder has its own README; consult those for software dependencies, script order, and inputs/outputs.
- Most tree files in `topologies/data/` are committed but absolute path references in `genome_snps_and_refs/*.txt` and the RADinitio shell scripts will not resolve outside their original environments.

## Conventions

- Tip labels in trees: `Quercus species|specimen_code` (stripped to species only for cross-dataset comparisons).
- Trees rooted on *Lithocarpus* (or *Castanopsis* for empirical data).
- Bootstrap replicate trees use `.bt###` suffix and are kept separate from ML trees.
- Date format in filenames: `YYYY_MM_DD` (e.g., `merged_calls_2025_06_27.vcf.gz`).
