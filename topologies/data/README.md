# data/

Input data for the topologies analysis pipeline. **These files are not generated by scripts in this repository** — they come from external RAxML and ipyrad runs and must be present before running the pipeline.

## Subdirectories

### `ref_RAxML_snps/`
Empirical RAD-seq trees assembled against reference genomes. Each subdirectory (`ref_alba_raxml`, `ref_glauca_raxml`, `ref_longispica_raxml`, `ref_rubra_raxml`, `ref_tomentella_raxml`, `ref_variabilis_raxml`, `ref_virginiana_raxml`, `de_novo`) contains the standard RAxML output files for one reference genome:
- `RAxML_bipartitions.*` — ML tree with bootstrap support values on nodes (primary input to pipeline)
- `RAxML_bootstrap.*` — bootstrap replicate trees
- `RAxML_bestTree.*`, `RAxML_bipartitionsBranchLabels.*`, `RAxML_info.*` — auxiliary RAxML outputs

### `simulated_RAD/`
Trees from simulated RAD-seq data, structured identically to `ref_RAxML_snps/`. Four assemblies: `simRAD_Qalba_denovo_snps`, `simRAD_Qalba_ref_snps`, `simRAD_Qvar_denovo_snps`, `simRAD_Qvar_ref_snps`.

### `ReSeq_phylos/`
Phylogenies from pseudoreference whole-genome resequencing, one directory per reference genome (`reseq_Qalba`, `reseq_Qvar`). Same RAxML file structure as above.

### `ipyrad-stats/`
Assembly statistics from ipyrad runs, split into `empirical/` and `simulated/` subdirectories. Read by `scripts/05_ipyradStats.R`.

### `pseudoref-stats/`
VCF/variant calling statistics for the resequencing assemblies (`.stats` files). Read by `scripts/07_raxmlStats.R`.

## Metadata files

- `OakSpmTableExtract-322SingleTipPhylo-20250516.xlsx` — specimen metadata table used to label tree tips with taxonomy
- `Oak_wgs_sampling.xlsx` — whole-genome sequencing sampling information
- `tr.singletons.correlated.1.taxaGrepCrown.tre` — reference tree from the 2020 oak phylogeny study
